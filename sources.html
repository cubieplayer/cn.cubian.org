Linux

有两种获取源码的方式

  1. apt-get install linux-source
  2. git clone https://github.com/mmplayer/linux-sunxi.git -b dev/sunxi-3.4 --depth=1

Linux源码是位于 /usr/src/ 下的7z压缩包。可以使用p7zip -d 文件名解压。

头文件的包名称是 linux-headers。

内核配置

  • A10 硬件如 Cubieboard1 使用 config-cubian-base-sun4i
  • A20 硬件如 Cubieboard2，Cubietruck 使用 config-cubian-base-sun7i-3.4

编译指导

  • 配置
    make ARCH=arm menuconfig
  • 编译
    make -j4 ARCH=arm CROSS_COMPILE=arm-none-linux-gnueabi- uImage modules
  • 安装
    make STRIP=arm-none-linux-gnueabi-strip INSTALL_MOD_PATH=ROOTFS_PATH ARCH=arm INSTALL_MOD_STRIP=1 modules_install

U-Boot

分支信息

  • sunxi - A10,A20 MMC 引导
  • sunxi-nand - A10 NAND 引导，支持ext4命令
  • dev/a20-nand-ext4-v2 - A20 NAND 引导，支持ext4命令

编译指导

  • 从 sunxi 分支
    make Cubieboard|Cubieboard2|Cubietruck CROSS_COMPILE=arm-none-linux-gnueabi-
    会编译出 A10 和 A20 的硬件，如 Cubieboard 1，2 和 Cubietruck 的MMC引导程序u-boot
  • 从 sunxi-nand 分支
    make Cubieboard CROSS_COMPILE=arm-none-linux-gnueabi-
    会编译出 A10 的硬件，如 Cubieboard1 的NAND引导程序u-boot
  • 从 dev/a20-nand-ext4-v2 分支
    make sun7i CROSS_COMPILE=arm-none-linux-gnueabi-
    会编译出 A20 的硬件，如 Cubieboard2 和 Cubietruck 的 NAND 引导程序 u-boot
